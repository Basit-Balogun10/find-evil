# Find Evil Roadmap

This roadmap captures the implementation order we should follow before SIFT arrives and after it becomes available.

## Phase 0: Repository Publication

1. Keep the scaffold in a pushable state.
2. Exclude generated runtime logs from version control.
3. Create the GitHub remote and push the initial baseline.
4. Verify that the public repo matches the local state.

## Phase 1: Foundation Layer

1. Replace placeholder node bodies with real Layer 0 behavior.
2. Keep evidence integrity architectural, not prompt-based.
3. Make the audit trail immutable in spirit and traceable in practice.
4. Establish the lowest-level input/output contracts for all nodes.

## Phase 2: Core Analysis Layer

1. Implement memory analysis hooks.
2. Add threat-intelligence lookup adapters.
3. Distinguish active vs. dormant activity.
4. Build negative-space reasoning helpers.

## Phase 3: Cross-Referencing Layer

1. Correlate disk and memory results.
2. Map confirmed findings to MITRE ATT&CK.
3. Build the artifact relationship graph.
4. Calibrate confidence with explicit reasons.

## Phase 4: Higher-Order Reasoning Layer

1. Rank hypotheses and targeted checks.
2. Assess blast radius.
3. Add attacker-perspective reasoning.
4. Predict likely next steps.
5. Reconstruct the timeline.

## Phase 5: Output and Meta Layer

1. Decide when escalation is required.
2. Extract IOCs in structured form.
3. Benchmark accuracy against known data.
4. Produce analyst and executive reports.
5. Generate a remediation playbook.
6. Keep the self-correction loop capped and logged.

## Phase 6: Testing and Validation

1. Test graph wiring and routing.
2. Test max-iterations enforcement.
3. Test audit-log output and traceability.
4. Test evidence-boundary behavior with mocks.

## Phase 7: SIFT Integration

1. Add read-only adapters to the SIFT Workstation.
2. Replace stubs with real tool calls one node at a time.
3. Validate that destructive operations remain impossible.
4. Run sample cases and benchmark against ground truth.

## Phase 8: Submission Packaging

1. Expand the README with setup and try-it-out steps.
2. Add dataset documentation.
3. Add accuracy-report templates.
4. Produce architecture diagrams and execution-log examples.
5. Review the whole repo for competition completeness.
